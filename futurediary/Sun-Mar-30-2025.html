ADB

[ CS Android Linux ]

Android Debug Bridge is a very useful tool to debug android - in both wired and wireless ways .
use adbd daemon to start the daemon - the client [ android ] should have USB Debugging turned on.
Which would act like a client.
For the server part -

Basics [Genin]

  • Connect to the device physically using a USB cable or so. . .
  • Ensure its seriel number is listed in the adb devices command.
  • Enter adb tcpip 5555[or any port for that matter] to listen on your android.
    • This sets up a tcp/ip connection to your device , so now your can disconnect the physical connection to your device
  • run adb devices You will still see a seriel number for your devices - if not :
  • Enter adb connect <android_ipaddr>:<port_number>
    • You are now connected to your devices wirelessly.
  • Enter adb shell to access the shell of your device


Adb generally takes up port from 5554-5584.
Around 15 of them are emulators [ odd numbered ].
even numbered port for the consoles

Seriel Connection

  • You can also use the -soption to specify the seriel number.


  • To install a package that is present in your server :
    • adb -s <seriel_number> install <your_apk_name>.apk
  • If you are already in a shell just use adb install <your_apk_name>
  • Use -t for apks that your want to test More information
  • You can also have port fowarding adb forward <source_port&gr; <destination port>

Stuff

  • you can push and pull files using
    adb push <local><remote>
    adb push <remote><local>
    more info
  • exit or Ctrl + D to exit a adb shell session
  • -s is for a specific devices using its seriel number
    -d targets a single connected device using USB
    -e target an emulator running on your system
